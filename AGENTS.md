## 开发工作流程

### 核心原则
- **强制使用简体中文**: 所有 AI 回复、文档、注释、日志、提交信息必须使用简体中文
- **上下文优先**: 编码前必须分析现有实现，识别可复用的接口与约束
- **工具优先**: 优先使用 context7 查询文档，使用 desktop-commander 进行本地文件操作
- **质量优先**: 所有改动必须可编译、可验证，禁止 MVP 或占位符代码
- **标准化优先**: 优先复用官方 SDK、社区成熟方案，禁止自研方案

### 开发前必做（编码前强制检索）
1. **搜索相似实现**: 找到至少 3 个相似功能的实现，理解项目模式
2. **查询官方文档**: 使用 context7 查询涉及的库/框架的最新文档
3. **理解测试策略**: 分析现有测试代码，确定测试方式

### 代码质量标准
- **SOLID 原则**: 遵循单一职责、开闭原则、依赖倒置等设计原则
- **DRY 原则**: 避免重复代码，抽象共享逻辑为复用组件
- **简单性优先**: 每个函数/类单一职责，禁止过早抽象和炫技式写法
- **测试覆盖**: 提供单元测试，覆盖正常流程、边界条件和错误处理
- **代码风格**: 遵循项目既有风格，包括命名、格式化、导入顺序

### 工作流程

1. **理解需求**: 分析用户需求，明确目标和范围
2. **收集上下文**: 搜索相似实现，查询官方文档，理解项目模式
3. **设计方案**: 基于上下文设计实现方案，确定接口和测试策略
4. **实施编码**: 严格遵循 Coding 规范进行开发
5. **测试验证**: 编写测试用例，确保功能正确性
6. **代码审查**: 自查代码质量，确保符合所有规范
7. **编译验证**: 运行npm run lint进行自测
